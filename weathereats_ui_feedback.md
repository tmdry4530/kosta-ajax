# WeatherEats UI Feedback

## Overall Impression

WeatherEats의 UI는 따뜻한 색감, 카드 기반 레이아웃, 날씨/음식 이모지 활용 덕분에 첫인상이 친근하고 과제용 서비스로도 충분히 완성도가 높습니다.
특히 메인 → 검색 → 즐겨찾기 흐름이 직관적이라 사용자가 빠르게 목적을 이해할 수 있습니다.

---

## What Works Well

### 1. 컨셉이 바로 보인다
- 메인 화면에 날씨 카드 + 추천 메뉴 카드가 바로 보여서 서비스 목적이 명확합니다.
- 사용자가 "날씨 보고 메뉴 추천받는 서비스"라는 점을 즉시 이해할 수 있습니다.

### 2. 카드 기반 구조가 안정적이다
- 날씨 카드, 추천 카드, 검색 결과 카드, 즐겨찾기 카드가 모두 같은 톤으로 정리되어 있어서 UI 일관성이 좋습니다.
- hover / active / badge 구조도 비교적 자연스럽습니다.

### 3. 모바일 하단 탭바가 앱처럼 느껴진다
- 홈 / 검색 / 저장으로 바로 이동할 수 있어서 모바일 사용성이 좋습니다.
- 손가락 도달성 측면에서도 괜찮은 선택입니다.

### 4. 상태 메시지가 잘 보인다
- 검색 성공, 즐겨찾기 저장, 에러 메시지가 상단 배너로 보여서 피드백이 즉시 전달됩니다.
- 사용자가 “지금 무슨 일이 일어났는지” 이해하기 쉽습니다.

### 5. 검색 페이지 구조가 실용적이다
- 검색창 → 필터 → 결과 리스트 → 지도 흐름이 자연스럽습니다.
- 모바일에서 결과 리스트를 먼저 보여주는 점도 좋습니다.

---

## 개선하면 더 좋아질 점

### A. 메인 페이지 첫 화면 CTA 강조
**현재:**
- 추천 카드와 직접 검색 버튼이 보이긴 하지만, 첫 화면에서 가장 먼저 눌러야 할 액션의 강조가 아주 강하진 않습니다.

**개선 제안:**
- "추천 메뉴 보러가기" 혹은 "내 주변 맛집 찾기" 버튼을 hero 영역 안에도 한 번 더 두기
- 메인 hero에 1차 CTA를 더 크게 두면 전환이 좋아질 수 있음

### B. 검색 결과 카드의 정보 우선순위 정리
**현재:**
- 가게명, 카테고리, 거리, 주소, 전화번호, 버튼이 모두 들어가 있어 정보량이 조금 많게 느껴질 수 있습니다.

**개선 제안:**
- 모바일에서는
  1) 가게명
  2) 거리
  3) 카테고리
  4) 주소
  순서로 더 강하게 시각적 계층을 주기
- 전화번호는 없으면 숨기거나 더 작은 톤으로 처리

### C. 지도 영역의 존재감 조절
**현재:**
- 모바일에서는 결과 리스트가 먼저 보이는 게 좋지만, 지도 영역이 다소 "보조 영역"처럼 느껴질 수 있습니다.

**개선 제안:**
- 지도 상단에 "선택한 맛집 위치" 같은 작은 보조 타이틀 추가
- 리스트 카드 클릭 시 지도 영역이 더 명확히 반응하도록 강조하면 체감 품질이 올라감

### D. 즐겨찾기 페이지 empty/full 상태 대비 강화
**현재:**
- 빈 상태는 잘 보이지만, 저장된 항목이 많아졌을 때 약간 단조롭게 느껴질 수 있습니다.

**개선 제안:**
- 즐겨찾기 카드에 작은 순번 배지나 카테고리 칩 추가
- 저장 개수에 따라 상단 카운트를 조금 더 강조

### E. 필터칩 active 상태 대비 강화
**현재:**
- 활성 필터도 보이지만, 빠르게 훑을 때 시각적 차이가 아주 강한 편은 아닙니다.

**개선 제안:**
- active 상태에서 배경색 대비를 조금 더 강하게 주거나
- underline / shadow / icon 등을 추가해도 좋음

## Page-by-Page Feedback

## 1. Home (`index.html`)

### Good
- 날씨, 추천, 예보 3단 구성이 명확함
- 추천 카드가 바로 검색으로 이어지는 구조가 좋음
- 날씨 테마 배경이 컨셉과 잘 맞음

### Better
- hero 액션 버튼 1개 더 강조
- 추천 카드에 "왜 추천했는지" 한 줄을 조금 더 크게 보여줘도 좋음
- 날씨 카드 안에서 현재 위치 텍스트를 더 눈에 띄게 만들어도 좋음

---

## 2. Search (`search.html`)

### Good
- 검색/필터/리스트/지도 역할 구분이 명확함
- 페이지네이션까지 포함돼 있어서 과제 완성도가 높음
- 데모 모드/fallback 메시지도 잘 처리됨

### Better
- 카드 간 간격을 모바일에서 조금 더 넓게 주면 터치성이 좋아질 수 있음
- 거리 정보와 저장 버튼을 좀 더 시선이 가는 위치에 배치하면 좋음
- 지도 영역에 선택 상태가 더 강하게 드러나면 좋음

---

## 3. Favorites (`favorites.html`)

### Good
- 저장 목록 확인, 삭제, 검색 복귀 흐름이 분명함
- 카드 정보가 충분해서 다시 선택하기 쉬움

### Better
- 저장일 표시와 카테고리 정보를 칩 스타일로 바꾸면 더 보기 좋아질 수 있음
- 상단 카운트와 전체삭제 버튼 사이 위계 차이를 조금 더 주면 좋음

---

## Priority Suggestions

### High
1. 메인 hero CTA 강화
2. 검색 카드 정보 계층 정리
3. 필터 active 상태 시각 대비 강화

### Medium
4. 즐겨찾기 카드 시각 요소 보강
5. 지도 선택 상태 반응 강화

### Low
6. 카테고리 칩/배지 디테일 추가
7. 마이크로 인터랙션(transition) 조금 더 풍부하게

---

## One-line Summary

> 현재 UI는 과제 제출용으로 충분히 완성도 높고 흐름도 좋다. 다만 CTA 강조, 검색 카드 계층, 필터/지도 반응성을 조금만 더 다듬으면 훨씬 더 앱다운 인상을 줄 수 있다.
